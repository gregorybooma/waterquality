Timestamp Format

All tables uploaded to our database must include a column containing well-formed timestamps. Examples of valid formats are given below. Most importantly, the timestamp must be in MDY (month, day, year) mode or an ISO 8601 ambiguous format (see "in any mode" in examples). Using any other mode will corrupt your timestamp. To illustrate: 08/07/12 is Aug. 7, 2012 in MDY mode; July 8, 2012 in DMY mode; and July 12, 2008 in YMD mode. Please double-check your timestamp column before uploading your data.

Any combination of the date and time elements is acceptable, provided that the date is included and appears first. Time may be omitted, but note that this will set the time to midnight on the date specified.

Valid Date Formats

Example Description (MDY = month, day, year)
1999-01-08 ISO 8601; January 8 in any mode (recommended format)
January 8, 1999 unambiguous in any datestyle input mode
1/8/1999 January 8 in MDY mode; user *must* ensure MDY mode
01/02/03 January 2, 2003 in MDY mode; user *must* ensure MDY mode
1999-Jan-08 January 8 in any mode
Jan-08-1999 January 8 in any mode
08-Jan-1999 January 8 in any mode
08-Jan-99 January 8, error in YMD mode
Jan-08-99 January 8, error in YMD mode
19990108 ISO 8601; January 8, 1999 in any mode
990108 ISO 8601; January 8, 1999 in any mode
1999.008 year and day of year
J2451187 Julian day

Valid Time Formats

Example Description
04:05:06 ISO 8601 (recommended format)
04:05:06.789 ISO 8601
04:05 ISO 8601
040506 ISO 8601
04:05 AM same as 04:05; AM does not affect value
04:05 PM same as 16:05; input hour must be <= 12

Time Zone Support

For the sake of consistency and metadata, we require the user to select the reference time zone name as part of the upload process. In situations where it is impractical for the user to reformat the timestamp, time zone information is supported if it is valid (i.e. it will not cause an error), but it will be ignored in favor of the user-provided time zone. Accepted forms are as follows:

Example Description
PST Abbreviation (for Pacific Standard Time)
America/New_York Full time zone name
PST8PDT POSIX-style time zone specification
-8:00 ISO-8601 offset for PST
-800 ISO-8601 offset for PST
-8 ISO-8601 offset for PST
zulu Military abbreviation for UTC
z Short form of zulu
